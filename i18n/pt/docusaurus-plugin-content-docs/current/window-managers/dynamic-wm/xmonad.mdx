Xmonad é um gerenciador de janelas em mosaico para o X. As janelas são organizadas automaticamente para ocupar a tela sem lacunas ou sobreposição, maximizando o uso da tela. O Xmonad é escrito, configurado e extensível em Haskell. Algoritmos de layout personalizados, bindings de teclas e outras extensões podem ser escritos pelo usuário nos arquivos de configuração. Os layouts são aplicados dinamicamente, e diferentes layouts podem ser usados em cada espaço de trabalho. O Xinerama é totalmente suportado, permitindo que as janelas sejam organizadas em várias telas físicas.

:::

## Instalação

Após concluir a instalação do Archcraft, **Xmonad** pode ser instalado em cima dele. Ele já vem pré-configurado e pronto para uso.

<Tabs>

<TabItem value="wm_1" label="Regular" default>

Abra um terminal e execute o seguinte comando para instalar a versão regular do Xmonad:
```bash
$ sudo pacman -Sy archcraft-xmonad
```
</TabItem>

<TabItem value="wm_2" label="Premium" default>

Baixe o pacote premium de [ko-fi](https://ko-fi.com/s/3f5db76f5b) e salve-o em um diretório. Abra um terminal no mesmo diretório e execute o seguinte comando para instalar o Xmonad premium:
```bash
$ sudo pacman -U archcraft-xmonad-premium-X.Y-Z-any.pkg.tar.zst
```
</TabItem>

</Tabs>

:::danger[Archcraft Prime]

<Highlight color="#62aeef">Xmonad Premium</Highlight> não vem pré-instalado na [**`Archcraft Prime ISO`**](https://ko-fi.com/s/c925a2a8c1)

:::

## Configuração

O Xmonad já vem pré-configurado no Archcraft. Nesta seção, veremos como o gerenciador de janelas Xmonad é configurado, quais ferramentas e utilitários são usados para fazê-lo funcionar e se parecer com um ambiente de desktop completo.

### Estrutura de Configuração

A estrutura de configuração do Xmonad no Archcraft é semelhante a isto:

```
~/.config
└── xmonad          : Diretório de configuração do Xmonad
    ├── alacritty   : Configuração do terminal
    ├── scripts     : Scripts usados pelo Xmonad
    ├── themes      : Todos os temas (configurações do polybar, rofi, etc.)
    ├── dunstrc     : Configuração do dunst para notificações
    ├── picom.conf  : Configuração do compositor
    ├── xmonad.hs   : Arquivo principal de configuração do Xmonad
    └── xsettingsd  : Arquivo de configuração de temas GTK, ícones e fontes
```

### Terminal

**Alacritty** é o terminal padrão. No gerenciador de janelas Xmonad, o Alacritty é iniciado com um arquivo de configuração alternativo. Se precisar fazer alterações, certifique-se de modificar os arquivos de configuração no diretório `~/.config/xmonad/alacritty`. Os seguintes arquivos de configuração podem ser modificados para alterar o comportamento do seu terminal:

**`~/.config/xmonad/alacritty/alacritty.toml`** : Arquivo principal de configuração <br/>
**`~/.config/xmonad/alacritty/colors.toml`** : Configurações de cores <br/>
**`~/.config/xmonad/alacritty/fonts.toml`** : Configurações de fontes

> Se precisar de ajuda com a configuração do Alacritty, execute: `man 5 alacritty` no terminal.

:::warning

Quando você altera o `tema`, os arquivos **colors.toml** e **fonts.toml** são sobrescritos pelo script do tema.

> Isso é aplicável apenas no <Highlight color="#c778dd">Xmonad Premium</Highlight>, pois os temas estão disponíveis apenas nele. Na versão regular, há apenas um tema.

:::

### Notificação

**Dunst** é usado como o daemon de notificações para o Xmonad. No gerenciador de janelas Xmonad, o Dunst é iniciado com um arquivo de configuração alternativo, que é o `~/.config/xmonad/dunstrc`. Portanto, se você precisar modificar algo, certifique-se de editar este arquivo.

> Se precisar de ajuda com a configuração do Dunst, execute: `man 5 dunst` no terminal.

:::warning

Quando você altera o `tema`, as fontes e cores no arquivo **dunstrc** são sobrescritas pelo script do tema.

> Isso é aplicável apenas no <Highlight color="#c778dd">Xmonad Premium</Highlight>, pois os temas estão disponíveis apenas nele. Na versão regular, há apenas um tema.

:::

### Papel de Parede

**hsetroot** define o papel de parede no Xmonad. Se você deseja alterar o papel de parede, edite os seguintes scripts:

<Tabs>

<TabItem value="wm_1" label="Regular" default>

Edite o script `~/.config/xmonad/scripts/xmonad_autostart` e adicione o caminho para o seu papel de parede.
```bash
# Restaurar o papel de parede
hsetroot -cover '/caminho/para/seu/papel/de/parede'
```
</TabItem>

<TabItem value="wm_2" label="Premium" default>

Edite o script `~/.config/xmonad/themes/wallpaper.sh` e adicione o caminho para o seu papel de parede.
```bash
WALLPAPER='/caminho/para/seu/papel/de/parede'
```
:::warning

Quando você altera o `tema`, o script `wallpaper.sh` será sobrescrito pelo script do tema.

:::

</TabItem>

</Tabs>

### Barra de Status

**Polybar** é usado como a barra de status no gerenciador de janelas Xmonad do Archcraft.

<Tabs>

<TabItem value="wm_1" label="Regular" default>

As configurações do polybar podem ser encontradas no diretório _theme_ do Xmonad em `~/.config/xmonad/theme`.

Se você deseja modificar as configurações do **polybar**, ou talvez adicionar/remover módulos, você precisa editar os seguintes arquivos:

**`~/.config/xmonad/theme/polybar/config.ini`**: Arquivo de configuração principal <br/>
**`~/.config/xmonad/theme/polybar/colors.ini`**: Arquivo de configurações de cores <br/>
**`~/.config/xmonad/theme/polybar/modules.ini`**: Arquivo de configuração de módulos

> Se precisar de ajuda com a configuração do polybar, visite: [wiki do polybar](https://github.com/polybar/polybar/wiki)

</TabItem>

<TabItem value="wm_2" label="Premium" default>

As configurações do polybar podem ser encontradas no diretório do _tema atualmente aplicado_, que está dentro do diretório _themes_ do Xmonad em `~/.config/xmonad/themes`.

Digamos que o tema atualmente aplicado é `default` e você quer modificar as configurações do **polybar**, ou talvez adicionar/remover módulos, então você precisa editar os seguintes arquivos:

**`~/.config/xmonad/themes/default/polybar/config.ini`**: Arquivo de configuração principal <br/>
**`~/.config/xmonad/themes/default/polybar/colors.ini`**: Arquivo de configurações de cores <br/>
**`~/.config/xmonad/themes/default/polybar/modules.ini`**: Arquivo de configuração de módulos

> Se precisar de ajuda com a configuração do polybar, visite: [wiki do polybar](https://github.com/polybar/polybar/wiki)

:::warning

Quando você altera o `tema`, as fontes no arquivo **config.ini** e o arquivo **colors.ini** do polybar serão sobrescritos pelo script do tema.

:::

</TabItem>

</Tabs>

### Lançadores e Applets

**Rofi** é utilizado para lançadores de aplicativos, executador de comandos e vários applets. Para modificar a configuração do rofi, edite o arquivo `~/.config/rofi/config.rasi`.

> Se precisar de ajuda com a configuração do rofi, execute: `man rofi` no terminal.

Os scripts para o rofi utilizados no Xmonad podem ser encontrados no diretório `~/.config/xmonad/scripts`. Estes são os scripts que utilizam rofi para fornecer lançadores e applets no Xmonad:

**`rofi_askpass`**: frontend rofi para `sudo`, utilizado para obter a senha de root <br/>
**`rofi_asroot`**: applet para abrir aplicativos como root <br/>
**`rofi_bluetooth`**: applet de Bluetooth para conectar rapidamente a dispositivos BT <br/>
**`rofi_launcher`**: lançador de aplicativos, executador de comandos, gerenciador de arquivos e gerenciador de tarefas <br/>
**`rofi_music`**: applet de música, que controla o MPD com `mpc` <br/>
**`rofi_powermenu`**: menu de energia, com diálogo de confirmação <br/>
**`rofi_runner`**: permite rapidamente executar um comando <br/>
**`rofi_screenshot`**: applet de captura de tela que funciona com o maim <br/>
**`rofi_themes`**: applet para mudar os temas **`somente no premium`** <br/>
**`rofi_windows`**: applet para gerenciar janelas abertas <br/>

Se você quiser modificar algum applet ou talvez estender a funcionalidade de algum deles, basta editar os scripts mencionados acima.

<Tabs>

<TabItem value="wm_1" label="Regular" default>

A configuração de cada script pode ser encontrada no diretório _theme_ do Xmonad em `~/.config/xmonad/theme`.

Vamos supor que você queira modificar a aparência do **launcher**, então você precisa editar o arquivo `~/.config/xmonad/theme/rofi/launcher.rasi`. O mesmo vale para qualquer outro applet, basta editar o respectivo arquivo no mesmo diretório.

> Se precisar de ajuda com a personalização do rofi, execute: `man rofi-theme` no terminal.

Para alterar as cores e fontes dos lançadores e applets, edite os arquivos `~/.config/xmonad/theme/rofi/shared/colors.rasi` e `~/.config/xmonad/theme/rofi/shared/fonts.rasi`.

</TabItem>

<TabItem value="wm_2" label="Premium" default>

A configuração de cada script pode ser encontrada no diretório do _tema atualmente aplicado_, que está dentro do diretório _themes_ do Xmonad em `~/.config/xmonad/themes`.

Vamos supor que o tema atualmente aplicado seja `default` e você queira modificar a aparência do **launcher**, então você precisa editar o arquivo `~/.config/xmonad/themes/default/rofi/launcher.rasi`. O mesmo vale para qualquer outro applet, basta editar o respectivo arquivo no mesmo diretório.

> Se precisar de ajuda com a personalização do rofi, execute: `man rofi-theme` no terminal.

Para alterar as cores e fontes dos lançadores e applets, edite os arquivos `~/.config/xmonad/themes/<THEME>/rofi/shared/colors.rasi` e `~/.config/xmonad/themes/<THEME>/rofi/shared/fonts.rasi`.

:::warning

Quando você altera o `tema`, os arquivos **shared/colors.rasi** e **shared/fonts.rasi** serão sobrescritos pelo script do tema.

:::

</TabItem>

</Tabs>

### Compositor

**Picom** é utilizado para composição. É um compositor leve com sombras, desfoque avançado e fading. No Xmonad, o picom é iniciado com um arquivo de configuração alternativo, que é o `~/.config/xmonad/picom.conf`.

Se precisar alterar algo relacionado ao picom, edite este arquivo de configuração do picom.

> Se precisar de ajuda com a configuração do picom, consulte o arquivo de configuração padrão do picom em `/etc/xdg/picom.conf`.

:::info

Se você estiver enfrentando problemas de tearing de tela, usar o backend `glx` pode resolver esse problema. <br/>
Se você estiver enfrentando lentidão, travamentos ou congelamentos, tente alternar entre os backends `xrender` e `glx`.  

:::

:::warning

Quando você altera o `tema`, configurações como _backend_, _sombras_, e _desfoque_ no arquivo **picom.conf** serão sobrescritas pelo script do tema.

> Isso se aplica apenas ao <Highlight color="#c778dd">Xmonad Premium</Highlight>, já que os temas estão disponíveis apenas nessa versão. Na versão regular, existe apenas um tema.

:::

### Window Manager

A configuração do Xmonad está disponível em `~/.config/xmonad`. Para configurar o gerenciador de janelas Xmonad, este é o arquivo principal usado:

**`~/.config/xmonad/xmonad.hs`**: arquivo principal de configuração do Xmonad, com configurações gerais, keybindings, layouts e regras de janelas.<br/>

> Se precisar de ajuda com a configuração do xmonad, visite: [Documentação do Xmonad](https://xmonad.org/documentation.html)

:::warning

Quando você altera o `tema`, configurações como _cores da borda_, _largura da borda_, no arquivo **xmonad.hs** serão sobrescritas pelo script do tema.

> Isso se aplica apenas ao <Highlight color="#c778dd">Xmonad Premium</Highlight>, já que os temas estão disponíveis apenas nessa versão. Na versão regular, existe apenas um tema.

:::

## FAQs

<details>
<summary><b>How to update my config to new configs?</b></summary>

Quando uma nova versão do Archcraft Xmonad é lançada, e você atualiza o sistema usando `sudo pacman -Syu` ou atualiza o pacote do Xmonad com `sudo pacman -S archcraft-xmonad`, novos arquivos de configuração são instalados como **xmonad_pacnew_YYYY-MM-DD** no diretório `~/.config`.

Para usar as novas configurações, faça um backup do seu diretório antigo `~/.config/xmonad` e mova o diretório `~/.config/xmonad_pacnew_YYYY-MM-DD` para `~/.config/xmonad`, e faça login novamente.

> Para o `Xmonad Premium`, baixe o pacote atualizado no Ko-fi (gratuitamente) e instale-o para aplicar as novas configurações.
</details>

<details>
<summary><b>Como mudar o tema GTK, ícones e cursor?</b></summary>

Para mudar o tema GTK, os ícones e o cursor na sessão Xmonad, edite o arquivo `~/.config/xmonad/xsettingsd` e reinicie o Xmonad para aplicar as alterações.

```bash
Net/ThemeName "Rick"
Net/IconThemeName "Zafiro-Red"
Gtk/CursorThemeName "Qogirr-Dark"
```

</details>

<details>
<summary><b>Como gerenciar múltiplos monitores e resolução no Xmonad?</b></summary>

Se você usa múltiplos monitores com o gerenciador de janelas Xmonad, pode gerenciá-los executando o aplicativo _Manager Monitors_ ou o comando `archcraft-randr`.
</details>

<details>
<summary><b>Por que o Alacritty não está funcionando no meu computador?</b></summary>

Como o Alacritty é um emulador de terminal baseado em OpenGL, ele pode não funcionar em alguns hardwares de computador. Você pode fazê-lo funcionar em tais computadores seguindo os seguintes métodos:

1. Abra o Alacritty com a variável **LIBGL_ALWAYS_SOFTWARE=1**. Edite as teclas de atalho ou o script que inicia o Alacritty e substitua `alacritty` por `LIBGL_ALWAYS_SOFTWARE=1 alacritty`.
2. Edite o arquivo de desktop **/usr/share/applications/Alacritty.desktop** e substitua `Exec=alacritty` por `Exec=env LIBGL_ALWAYS_SOFTWARE=1 alacritty`.

</details>
